# 기획서: 채널별 요약 탭 — 채널 다중 선택

- 작성일: 2026-07-08
- 작성자: yennie7120 (Maintainer)
- 상태: 기획 검토 중 (구현 전)
- 관련 화면: 메인 탭 "채널별 요약" (`src/components/MdSummarySection.tsx`)

## 1. 현재 동작

`MdSummarySection`의 채널 탭 바(`TABS = ['전체 요약', ...CHANNELS]`)는 **단일 선택(라디오형)**이다.

- `activeTab: '전체 요약' | Channel` — 한 번에 하나만 활성화
- `'전체 요약'` 선택 시 → `MdSummaryOverview`: 8개 채널(`자사몰·스스·위탁·쿠팡·B2B·사입및페어·글로벌·일본`)을 모두 합산/비교하는 뷰
- 특정 채널 1개 선택 시 → `MdChannelDetail`: 그 채널 하나만의 월별 매출·공헌이익 상세 뷰

즉 지금은 "전체" 아니면 "채널 1개"만 가능하고, "쿠팡 + 자사몰만 묶어서 보기" 같은 부분 조합은 불가능하다.

## 2. 목표

채널 탭을 **다중 선택(체크형)**으로 바꿔서, 사용자가 원하는 채널 조합(0개=전체, 1개, N개)을 자유롭게 골라 그 조합의 합산 수치를 볼 수 있게 한다.

예: B2C 채널만(`자사몰+스스+위탁+쿠팡`) 묶어보기, 해외 채널만(`글로벌+일본`) 묶어보기 등.

## 3. 구현 방법 검토

### 방법 A — 탭 바를 체크 토글로 전환 (추천)

기존 탭 바 UI(버튼 pill)를 그대로 두고 클릭 동작만 "선택 전환(토글)"으로 바꾼다. 채널 수가 8개뿐이라 드롭다운보다 한눈에 보이는 pill 방식이 적합.

- 상태: `activeTab: TabId` (단일) → `selectedChannels: Set<Channel>` (다중)
- `selectedChannels.size === 0` → 현재의 "전체 요약"과 동일 (모든 채널 비교 뷰)
- `selectedChannels.size >= 1` → 선택된 채널들만 합산한 상세 뷰
- "전체 요약" pill은 "선택 초기화(전체 보기로 리셋)" 버튼 역할로 유지
- 장점: 이미 있는 SKU 필터 드롭다운(체크박스 다중선택, `MdSummarySection.tsx:269-332`)과 인터랙션 패턴이 통일됨. 코드 변경 범위가 가장 작음.

### 방법 B — 별도 드롭다운으로 채널 다중선택 분리

탭 바(전체/채널 상세 모드 전환용)는 그대로 두고, SKU 드롭다운 옆에 "채널 필터" 드롭다운을 추가해 다중 채널을 고르게 함.

- 장점: 기존 "전체 요약 vs 채널 상세" 개념을 안 건드림
- 단점: 개념이 두 겹(탭 모드 + 채널 필터)이 되어 사용자가 헷갈릴 수 있음. UI 요소도 하나 더 늘어남.

### 방법 C — 탭 바는 유지, 채널 상세 뷰 안에 다중선택 체크박스 추가

'전체 요약'을 클릭했을 때만 그 안에서 채널을 체크박스로 골라 부분합을 보게 함.

- 단점: "채널 탭을 눌러서 채널을 보는" 기존 직관과 어긋남(전체 요약 안에서만 다중선택이 되는 게 부자연스러움)

**결론: 방법 A 채택.** 기존 탭 UI의 시각적 위치를 그대로 유지하면서 클릭 시맨틱만 라디오→체크로 바꾸는 것이 변경 범위도 작고 사용자 학습 비용도 가장 낮음.

## 4. 상세 설계 (방법 A)

### 4.1 상태 모델

```ts
// MdSummarySection.tsx
const [selectedChannels, setSelectedChannels] = useState<Set<Channel>>(new Set());

function toggleChannel(ch: Channel) {
  const next = new Set(selectedChannels);
  next.has(ch) ? next.delete(ch) : next.add(ch);
  setSelectedChannels(next);
}
```

- 카테고리(`categoryFilter`)나 `activeBrand`가 바뀌면 기존처럼 선택을 초기화할지는 확인 필요 (현재 `selectedSkuIds`는 초기화됨 — `useEffect` at line 201). 채널 선택도 동일하게 초기화하는 쪽이 일관적.

### 4.2 탭 바 UI

- 각 채널 pill: `selectedChannels.has(ch)` 여부로 활성 스타일(현재의 `bg-indigo-600 text-white`) 적용 — 여러 개 동시에 활성 가능
- "전체 요약" pill: `selectedChannels.size === 0`일 때만 활성 스타일. 클릭 시 `setSelectedChannels(new Set())`

### 4.3 데이터 계산 재사용

다행히 계산 유틸(`src/utils/mdSummaryCalc.ts`)이 이미 채널 단위 함수 + `addMetrics` 합산 구조라 다중 채널 합산이 쉽다.

- `calcSkuAllChannelTotals(sku, months, ...)` — 현재 `CHANNELS`(8개) 전체를 하드코딩해서 순회 후 합산 (line 87)
- 이를 `calcSkuAllChannelTotals(sku, months, varCostMap, usdRate, jpyRate, channels = CHANNELS)` 처럼 **채널 목록을 선택적 파라미터로 일반화**하면, 다중 채널 합산에도 그대로 재사용 가능
- `MdChannelDetail.tsx`는 현재 `channel: Channel` 단일 prop을 받아 `calcChannelMonthMetrics(sku, channel, ...)`를 직접 호출하는 구조 → `channels: Channel[]` prop으로 바꾸고, 내부에서 `channels.reduce(...)`로 합산하도록 수정
- `MdSummaryOverview.tsx`의 `buildMonthlyChartData`도 `CHANNELS`를 하드코딩 순회(line 38) 중이므로 동일한 방식으로 일반화 가능 (다만 "전체 요약"은 선택이 없을 때만 쓰이므로 필수 변경은 아님)

### 4.4 뷰 분기

```ts
{selectedChannels.size === 0 ? (
  <MdSummaryOverview skus={filtered} months={visibleMonths} ... />
) : (
  <MdChannelDetail skus={filtered} channels={[...selectedChannels]} months={visibleMonths} ... />
)}
```

- `MdChannelDetail` 상단 타이틀/뱃지에 선택된 채널명을 모두 나열 (예: "쿠팡 + 자사몰 상세") — 채널이 많아지면 `2개 채널 선택` 처럼 축약 표시 고려

### 4.5 영향 파일

| 파일 | 변경 내용 |
|---|---|
| `src/components/MdSummarySection.tsx` | 상태를 `activeTab`(단일) → `selectedChannels`(Set) 로 전환, 탭 pill 클릭 핸들러를 토글로 변경 |
| `src/components/MdChannelDetail.tsx` | `channel: Channel` prop → `channels: Channel[]` prop, 내부 계산을 합산 방식으로 변경, 타이틀 표시 로직 수정 |
| `src/utils/mdSummaryCalc.ts` | `calcSkuAllChannelTotals`(및 필요 시 `calcSkuChannelTotals` 활용부)에 채널 목록 선택적 파라미터 추가 |
| `src/components/MdSummaryOverview.tsx` | (선택) 동일한 일반화 패턴 적용 시 재사용성 향상, 필수는 아님 |

### 4.6 엣지 케이스

- 선택된 채널 중 수량이 0인 채널만 있는 경우 → 기존 "채널 비중을 먼저 설정해주세요" 류 빈 상태 메시지 재사용
- 채널 8개를 전부 선택한 경우 → "전체 요약"과 수치는 동일하지만 뷰(비교 차트 vs 합산 상세)는 다름. 이 차이를 사용자에게 혼란 없이 전달할지 검토 (예: 8개 다 선택 시 자동으로 "전체 요약"으로 스냅할지 여부 — 일단은 사용자가 명시적으로 고른 채널 합산 뷰를 그대로 보여주는 쪽을 기본으로 제안)
- 세션 중 페이지 새로고침 시 선택 초기화됨 — 현재 `activeTab`도 세션 저장 안 하므로 동일 동작 유지(범위 밖)
